# 90坦克大战 Web 开发记录（v4）

日期：2026-03-30

## 本轮目标

- 关卡结构与校验达到“可验收”的严格标准：主基地底部贴底、基地三面砖保护、玩家出生点与所有地形（含草/冰）无重合。
- 移除未达验收标准的音频实现（后续专项再做）。
- 移除画布主题切换（游戏本体不需要支持主题；固定一套配色）。
- 调整 Debug 面板 UI：从 canvas 内覆盖层移到独立布局，且与主状态面板对齐、单行按钮布局。

## 本轮结论（代码层）

### 1) 基地与出生点严格对齐（Stage 2–10 已重点验收）

- 主基地（老鹰）像素位置：`base.y + base.size === WORLD_H`，底部与屏幕底边贴齐。
- 基地保护：仅在老鹰的 **上 + 左 + 右** 三面铺 `brick(B)`（底部朝玩家方向保持开放），并通过运行时/校验脚本强制一致。
- 玩家出生点：
  - 出生搜索时要求坦克整体不得“落入基地敞开方向的底部band”（使用 `BASE_TOP_Y` 排除）。
  - 出生矩形必须与 **任何非空地形** 不重合（砖/钢/水/草/冰都算）。

验证：

- `npm run validate:tank90`：通过。

### 2) 移除音频

- 删除 `src/tank90/audio.ts`。
- 移除 `audioCues` 相关的 `core/types.ts` / `core/state.ts` / `core/update.ts` / `TankBattle90View.tsx` 接入。
- `plan.md` 同步更新为“后续专项再实现”。

### 3) 固定画布配色（移除 Canvas 主题）

- 将 `src/tank90/core/palette.ts` 改为仅导出固定 `RENDER_PALETTE`，不再支持 Classic/Arcade 切换。
- `TankBattle90View.tsx` 删除 localStorage 主题切换与按钮。
- `e2e/tank90.spec.ts` 中对应测试用例更新为“不再验证主题切换，仅验证 canvas 尺寸与页面 shell 行为”。

### 4) Debug 面板布局调整

- Debug 面板由 canvas 覆盖层移出，放到“主状态(STAGE/按钮) 面板”和“游戏画布面板”之间。
- 按钮容器改为 `flex-nowrap`，保证单行。
- 调整 padding/对齐方式，使其与上方 `STAGE` 文本左侧对齐，且去掉不必要的标题占行。
